Movie Webpage Project

Context / Prompt

My teacher, Mr Mueller wanted me and a partner to work on a movie web-page project to practice our skills and everything we learned about basic HTML & CSS.

Content / Challenges

As briefly stated in the context / prompt section, Mr Mueller wanted me and a partner to work on a movie web-page project to put everything we've learned about basic HTML and CSS to test. This ranged from headers, paragraphs, divs, spans, fonts, element class and id, as well as importing local images and importing text fonts.

At first I thought the project was busy work, but during the process I realized it was more than that. My partner and I struggled with planning and had no idea where to start. Eventually we came up with the idea of a Mini Pekka going through life and eventually becoming a Pekka.

One personal issue I faced was optimizing the webpage so it looked visually appealing. I couldn’t figure out how to center images, text, or deal with divs and spans. Looking back at freeCodeCamp lessons and asking my partner for help helped me understand how divs and spans worked and how to organize the site properly.

Collaboration and research were huge parts of this project. The website W3Schools that Mr Mueller recommended helped a lot when learning how certain HTML elements worked.

Takeaways

This project refreshed many HTML and CSS fundamentals, but the biggest lessons were collaboration and planning. Without proper planning the project becomes messy. Planning actually took the majority of our time because we had to restart the project twice.

Mr Mueller had us create a plan.md which organized ideas like the plot, characters, settings, fonts, and overall theme. This helped us structure the project before writing code.

Collaboration was also extremely important. Two people working together gave different perspectives and helped solve problems faster.

Next Steps

If I had more time I would improve the layout and make the site more visually appealing. Some parts of the code could be cleaner and the design could be improved. Even though it's not perfect, the experience taught me a lot about planning, collaboration, and basic web development.
